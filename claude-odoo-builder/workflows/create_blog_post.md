# Create Blog Post Workflow

## Objective

Create and publish a blog post on the 19 Prince Odoo website using the `blog.post` model.

> **Do NOT use `website.page` or `push_page.py` for blog content.** Blog posts are a separate model
> (`blog.post`) and will not appear in Website > Blog Post Pages if pushed as a `website.page`.

---

## Key Facts (discovered 2026-05-14)

- Blog posts live in model: `blog.blog` (the blog) → `blog.post` (each post)
- Production blog ID: **1** (name: "Blog")
- URL pattern is auto-generated by Odoo: `/blog/blog-1/[slug]-[post-id]`
  - e.g. `/blog/blog-1/why-most-odoo-partners-can-t-answer-who-are-you-for-13`
  - You cannot set a custom URL slug — Odoo generates it from the title + ID
- Content field is raw HTML (your section content, no QWeb wrapping needed)
- Publishing requires setting BOTH `is_published` and `website_published` to `True`

---

## Required Inputs

- HTML content file (e.g., `.tmp/draft_my-post.html`)
- Post title (string)
- `.env` populated with production credentials: `PROD_ODOO_URL`, `PROD_ODOO_DB`, `PROD_ODOO_USER`, `PROD_ODOO_PASSWORD`

---

## Pre-Flight Checks

### 1. Verify production credentials

```bash
grep -E "^(PROD_ODOO_URL|PROD_ODOO_DB|PROD_ODOO_USER|PROD_ODOO_PASSWORD)=" .env | sed 's/=.*/=✓/'
```

Stop if any are missing.

### 2. Validate the HTML

```bash
python3 tools/validate_html.py --input .tmp/draft_my-post.html
```

Fix any errors before proceeding. Warnings are acceptable.

### 3. Confirm the blog ID

```python
python3 - <<'EOF'
import os, sys
sys.path.insert(0, 'tools')
from dotenv import load_dotenv
load_dotenv('.env')
from odoo_client import OdooClient

client = OdooClient(
    url=os.getenv("PROD_ODOO_URL"),
    db=os.getenv("PROD_ODOO_DB"),
    user=os.getenv("PROD_ODOO_USER"),
    password=os.getenv("PROD_ODOO_PASSWORD"),
)
blogs = client.search_read("blog.blog", [], ["id", "name"])
print("Blogs:", blogs)
EOF
```

Expected: `[{'id': 1, 'name': 'Blog'}]`. If the ID differs, update it in the steps below.

## Step 1: Create the Blog Post (Unpublished)

```python
python3 - <<'EOF'
import os, sys
sys.path.insert(0, 'tools')
from dotenv import load_dotenv
load_dotenv('.env')
from odoo_client import OdooClient

client = OdooClient(
    url=os.getenv("PROD_ODOO_URL"),
    db=os.getenv("PROD_ODOO_DB"),
    user=os.getenv("PROD_ODOO_USER"),
    password=os.getenv("PROD_ODOO_PASSWORD"),
)

with open(".tmp/draft_my-post.html") as f:
    content = f.read().strip()

post_id = client.create("blog.post", {
    "name": "Your Post Title Here",
    "blog_id": 1,
    "content": content,
    "is_published": False,
    "website_published": False,
})
print(f"Created blog.post ID: {post_id}")

posts = client.search_read("blog.post", [("id","=",post_id)], ["id","name","website_url"])
print("Post URL:", posts[0]["website_url"])
EOF
```

Note the `website_url` output — this is the URL Odoo assigned.

## Step 3: Publish

```python
python3 - <<'EOF'
import os, sys
sys.path.insert(0, 'tools')
from dotenv import load_dotenv
load_dotenv('.env')
from odoo_client import OdooClient

client = OdooClient(
    url=os.getenv("PROD_ODOO_URL"),
    db=os.getenv("PROD_ODOO_DB"),
    user=os.getenv("PROD_ODOO_USER"),
    password=os.getenv("PROD_ODOO_PASSWORD"),
)

POST_ID = 0  # replace with actual post ID from Step 1

client.write("blog.post", [POST_ID], {
    "is_published": True,
    "website_published": True,
})
print(f"Published blog.post ID {POST_ID}")
EOF
```

---

## Step 4: Verify in Odoo Backend

Go to **Website > Blog Post Pages** and confirm the post appears with "Is Published" checked.

---

## Rollback / Unpublish

To take the post down without deleting it:

```python
client.write("blog.post", [POST_ID], {
    "is_published": False,
    "website_published": False,
})
```

To delete entirely (use with caution):

```python
client.unlink("blog.post", [POST_ID])
```

---

## Staging First (Recommended)

To test on staging before production, swap `PROD_ODOO_*` credentials for `ODOO_*` (staging) credentials in all steps above. The blog ID may differ on staging — run the confirm step to check.

## Common Errors

| Error | Cause | Fix |
|---|---|---|
| Post not appearing in Blog Post Pages | Used `website.page` instead of `blog.post` | Deactivate the wrong page, recreate with `blog.post` |
| Content looks unstyled | HTML references external CSS not loaded in blog template | Use Bootstrap 5 utility classes only |
| URL slug is wrong/ugly | Odoo generates slug from title — can't be customized via API | Edit the slug manually in Odoo backend after creation |
| `is_published` set but post still hidden | Need to set BOTH `is_published` AND `website_published` | Set both fields to `True` |
